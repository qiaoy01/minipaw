# Minipaw 研究方向工作文档（7人小组）

## 背景

Minipaw 是双模型系统, 将小型LLM与大型"顾问"LLM 连接起来。三阶段架构——训练 → 试用 → 工作——逐步建立小模型独立处理与路由给顾问的明确边界。

当前三类核心问题驱动本次研究：

1. **小型 LLM 的幻觉问题** 
2. **Advisor 模式架构**——如何让训练、路由和边界学习端到端生效
3. **多步执行健壮性**

---

## 任务簇

### A. 幻觉抑制

**A1 — 标签化输出（H1）**
小模型把多个值塞进同一个print()调用，事后追溯时编造中间值。
- 在 `prompts/minihow.md` 添加规则：每个独立有意义的量必须单独出现在带标签的输出行（如 `print(f'hour={h}')`），再打印合并结果。
- 构造 20 个触发多值场景的用例，对比加规则前后的追溯准确率。
- 交付物：规则补丁 + 评测脚本 + 准确率前后对比表。
- 指标：计算后中间值正确召回率（%）。

**A2 — 上下文窗口与"已确认执行"锚点（H2）**
`CONTEXT_MAX_BYTES=2048` 截断旧轮次；模型声称从未写过它确实写过的文件。
- （1）将上限提升至 4096/8192，测量 token 开销与召回收益。
- （2）在任何实际之后，注入一条优先保留的相关的紧凑记录：`[confirmed] wrote <value> to <path>`。
- 指标：5 步以上任务中"忘记已写入"错误发生率。

**A3 — DONE 前回读校验（H3）**
发出 DONE 后，模型从记忆中重构算术，产生内部不一致的解释。
- 加强 `prompts/minihow.md` Rule 8："在 DONE 中解释计算时，只能使用先前 EXEC 结果中明确出现的数字。"
- 对写-读-解释链路强制要求在 DONE 前进行一次回读。
- 复现 `2000 + 1777924904 + 16 = 1777926904` 类不一致案例集。
- 交付物：失败用例集 + prompt 加固 + 修复率。
- 指标：DONE 段算术内部一致率。

### B. Advisor 三阶段架构

**B1 — 调整有效性度量（训练阶段）**
`adjust-meta.md` 输出调整建议，但没有指标证明小模型调整后真的趋近大模型。
- 从三个维度定义一致性评分：语义、结构、值精度。
- 抽取 50 个任务跑"调整前 → 调整中 → 调整后"三轮，绘制每轮趋同率曲线。
- 交付物：评估器模块 + 30 轮调整趋势图。
- 指标：每轮调整循环趋同率提升幅度。

**B2 — 路由表设计（试用阶段）**
默认走小模型，用户标注"哪类任务调顾问"——这个标注与路由表需要持久化并生效。
- 设计任务分类体系：`{minihow, miniwhy, miniwhat}` × 风险等级 × 历史成功率。
- 实现路由表持久化与覆盖规则。
- 交付物：路由表 schema + CLI 命令 + 集成测试。

**B3 — 边界自动学习（工作阶段）**
人工标注路由成本高。试用阶段两模型的一致性日志可以驱动边界自动发现。
- 按特征对试用日志做聚类：任务长度、是否含数值、工具调用次数、关键词。
- 训练一个轻量门控规则（纯规则，不上模型），自动分类"走顾问"vs"走小模型"。
- 交付物：门控规则文件 + 精确率/召回率报告。

### C. 多步执行健壮性

**C1 — 链式幻觉防护（TC-M1）**
经过 3 步真实验证后，DONE 段又虚构 4 级父进程链。
- 实现 DONE 段覆盖率校验器：每条事实声明必须能定位到某条 EXEC 输出（字符串覆盖启发式）。
- 校验不通过则强制再进行一轮核实，再接受 DONE。
- 交付物：校验器模块 + 阻断率/误伤率数据。

**C2 — MAX_SESSION_STEPS 动态预算（TC-M5）**
8 步上限太低，5 项子目标任务走不完。
- 让规划器在第 1 步申报子目标数量，据此动态分配步数预算。
- 交付物：`minicore.rs` 预算分配机制 + TC-M5 回归通过记录。

**C3 — 跨步工作记忆显式锚点**
将 H2 的"confirmed write"思路扩展到所有重要中间值：PID、文件名、URL。
- 识别哪些 EXEC 输出应被提升为持久记忆条目。
- 实现两条线对比：规则版提升 vs LLM 分类提升。
- 交付物：提升规则说明 + 两方案准确率对比。

---

### D. 路由与工具访问

**D1 — miniwhat 路径的技能感知（测试报告问题 1-4）**
"What is the current time?" 走 miniwhat，但技能不可见，查询失败。
- 在 miniwhat 入口先扫描 `skills/`，命中则降级回 minihow。
- 交付物：`minicore.rs` patch + 回归测试用例集。

**D2 — 分类边界优化（事实 vs 需执行才知的事实）**
分类器的 few-shot 示例对边界情形覆盖不足。
- 构建 100 条边界样本集，对比 prompt 优化前后的分类准确率。
- 交付物：样本集 + prompt 补丁 + 混淆矩阵。

**D3 — 工作区路径策略统一**
LLM 生成绝对路径被沙盒拒绝。
- 决策：要么在 prompt 中明确禁止绝对路径，要么让 ToolRunner 透明地将工作区内绝对路径重写为相对路径。
- 实现所选方案的完整链路。
- 交付物：决策说明 + 实现代码 + 回归测试。

---

### E. 元认知

**E1 — 显式不确定性声明（TC-B8 + 论文 §3.5/5.6）**
模型以不合理的确定性作答，未验证的声明与已验证的声明无法区分。
- 在 minihow/miniwhy prompt 添加："未经 EXEC 结果验证的论断必须前缀 `[unverified]`。"
- 运行 30 个评测用例。
- 交付物：prompt 补丁 + `[unverified]` 标注的精确率/召回率。

**E2 — 信心-正确率校准（论文 §7.2）**
- 在 DONE 时让 LLM 输出 0-100 的信心分。
- 跨任务收集，计算校准曲线（Brier score / ECE）。
- 交付物：评测脚本 + 校准曲线图。

| 同学 7 | E2 | 独立可并行的评测工作 |

## 高 ROI 优先级（若只能做 2-3 项）

**A2 → C1 → B1**

- **A2**（上下文锚点）：影响最大，纯配置 + 一条注入规则，修复最高频失败模式。
- **C1**（链式幻觉校验器）：直接提升多步输出的用户可信度。
- **B1**（advisor 评估器）：闭合 advisor 训练环路；没有趋同指标，训练过程无法验证。

**核心原则：脚手架补纪律（H1/H2/H3、C 簇），advisor 补能力上限，两层不可互相替代。**
